## 3. 최근 결정 사항

1. **SPA 아키텍처 및 include 패턴 채택 (2026-07-04)**
   - 학교 PC/태블릿 환경에서 원활히 동작하면서도 로컬 개발 협업을 편하게 유지하기 위해, Vite 등의 과도한 번들러 환경 대신 순수 HTML/JS 파일 분할 개발 후 GAS `include` 엔진을 활용한 단일 페이지 SPA 빌드 방식을 적용하기로 결정했습니다.
2. **동시 저장 충돌 대책으로 LockService 강제 적용 (2026-07-04)**
   - 여러 명의 학생이 한 수업 시간에 동시에 점수를 제출할 경우 데이터가 덮어씌워지거나 저장 실패가 일어나는 상황을 예방하기 위해, 모든 Sheets 기록 API 호출에 `LockService.getScriptLock()`을 최소 15초 한도로 사용하도록 규정했습니다.
3. **Canvas와 SVG 혼용을 통한 퍼즐 캔버스 최적화 (2026-07-04)**
   - 삼각 판타그램 조각 맞추기 등 단순 드래그가 주가 되는 것은 SVG를 사용하고, 미러 레이저 반사각 계산 등 렌더링 최적화가 필요한 복합형 퍼즐은 Canvas를 활용하도록 개별 퍼즐의 확장성 구조(`PuzzleBase`)를 채택했습니다.

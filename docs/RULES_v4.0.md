# RULES.md
# AI 바이브 코딩 코딩 규칙 v4.0

## 목적

이 문서는 프로젝트 전반에 적용되는 코딩 규칙, 아키텍처 원칙, 품질 기준을 정의한다.

---

# 1. 의사결정 우선순위

1. 데이터 무결성
2. 기존 기능 보존
3. 안정성
4. 보안
5. 유지보수성
6. 성능
7. 생산성
8. UI 개선

상위 원칙과 충돌하면 하위 원칙을 포기한다.

---

# 2. 최소 수정 원칙

허용

- 버그 수정
- 기능 구현
- 테스트 보완

금지

- 무관한 리팩토링
- 함수명 대량 변경
- 파일 구조 재편
- 주석 삭제

기존 기능을 유지하는 방향을 우선 선택한다.

# 3. 아키텍처 원칙

UI → Service → Repository

## 단일 책임

하나의 함수는 하나의 역할만 담당한다.

## 상태 관리

상태 변경은 중앙 관리 구조를 통해 수행한다.

## 의존성

순환 참조를 금지한다.

원칙

# 6. Git 운영

## Commit 형식

feat:
fix:
refactor:
docs:
chore:

## 원칙

- 모든 작업은 Git으로 관리한다.
- 의미 있는 단위로 Commit한다.
- Commit 메시지는 변경 목적을 명확히 작성한다.
- 실험성 작업은 별도 브랜치를 권장한다.

# 11. 리팩토링 규칙

# 13. 하지 말아야 할 것

- 추측으로 수정

# 14. 최종 원칙

빠른 개발보다 오래 유지되는 개발을 선택한다.

작동하는 코드보다 안전한 코드를 우선한다.

새 기능보다 기존 기능 보호를 우선한다.
